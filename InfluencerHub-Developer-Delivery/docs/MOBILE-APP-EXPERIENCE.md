# Mobile App Experience — InfluencerHub

تجربة الجوال شبيهة بتطبيق أصلي (App-like) — عبر المتصفّح و PWA، دون تطبيق أصلي منفصل.

## 1. عناصر التجربة الأصلية

| العنصر | التنفيذ |
|-------|---------|
| **التنقّل السفلي** | شريط ثابت أسفل الشاشة، 4–5 وجهات، أيقونة + تسمية، نقطة إشعار حمراء. يظهر <768px فقط. |
| **الدرج الجانبي** | ينزلق من الجانب (RTL: من اليمين) مع scrim معتم؛ يفتح بزر القائمة ويُغلق بـ Esc/نقرة خارجية/اختيار عنصر. |
| **الشريط العلوي للجوال** | لاصق أعلى الشاشة (sticky)، يحمل زر القائمة + الشعار + عنوان الصفحة. |
| **المساحات الآمنة** | يحترم النوتش وشريط الإيماءات عبر `env(safe-area-inset-*)` و`dvh`. |
| **بلا قفزات تكبير** | خط الإدخال ≥ 16px يمنع تكبير iOS عند التركيز. |
| **قابلية التثبيت** | PWA: `display:standalone`، أيقونات، شاشة بداية، تشغيل بلا شريط عنوان. |

## 2. القابلية للتثبيت (PWA)

راجع [PWA-ARCHITECTURE](PWA-ARCHITECTURE.md). أهم النقاط:
- Manifest + Service Worker آمن (لا يخزّن بيانات مصادَقة/حسّاسة).
- بديل offline للتنقّل، وتحديث هادئ عند توفّر نسخة.
- `theme-color` بنفسجي، `apple-mobile-web-app-*` لـ iOS.

## 3. أنماط الإدخال

- `inputmode="email"` للبريد، لوحة مفاتيح مناسبة لكل حقل.
- `autocomplete` صحيح (`username`/`current-password`) لدعم مدير كلمات المرور.
- أزرار بعرض كامل (`btn-block`) في النماذج على الجوال.

## 4. الأداء المُدرَك

- انتقالات قصيرة (140–240ms) مع `cubic-bezier` واحد.
- `will-change: transform` على الدرج فقط.
- خطوط تُحمّل بـ `display=swap` و`preconnect`.
