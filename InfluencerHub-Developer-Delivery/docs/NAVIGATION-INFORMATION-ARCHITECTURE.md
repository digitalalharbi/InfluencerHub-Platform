# Navigation & Information Architecture — InfluencerHub

بنية المعلومات والتنقّل. **مصدر واحد**: `config/navigation.php` يولّد الشريط الجانبي والتنقّل السفلي للجوال عبر كل البوابات.

## 1. المكوّنات

| المكوّن | الدور |
|--------|-------|
| `config/navigation.php` | إعداد التنقّل المركزي لكل بوابة (agency/client/partner/creator) |
| `resources/views/components/app-nav.blade.php` | يرسم الشريط الجانبي من الإعداد (مجموعات + عناصر + أيقونات + شارات) |
| `resources/views/components/app-bottom-nav.blade.php` | التنقّل السفلي للجوال (4–5 وجهات `mobile => true`) |
| `resources/views/components/icon.blade.php` | أيقونات Lucide (SVG داخلي، لا إيموجي، لا اعتماد خارجي) |
| `app/Support/Navigation/NavigationBadges.php` | عدّادات حقيقية من PostgreSQL (مُنطّقة بالمستأجر) |

## 2. بنية عنصر التنقّل

```php
['key' => 'service_requests', 'icon' => 'inbox', 'route' => '/app/service-requests',
 'match' => 'app/service-requests*', 'badge' => 'service_requests',
 'desc' => 'service_requests', 'mobile' => true],
```

- `key` → التسمية من `navigation.items.KEY`
- `icon` → اسم أيقونة Lucide
- `route` / `match` → المسار + نمط تحديد التبويب النشِط (يدعم `?query`)
- `badge` → عدّاد حقيقي (اختياري)
- `mobile` → يظهر في التنقّل السفلي (بحد أقصى 5)
- `soon` → قيد البناء (غير قابل للنقر، بلا شارة)

## 3. خريطة البوابات

### الوكالة (agency)
| المجموعة | العناصر |
|---------|---------|
| الرئيسية | لوحة التحكم |
| العلاقات | العملاء · العلامات التجارية · الوكالات الخارجية |
| المبدعون | المؤثرون · صنّاع المحتوى · كل المبدعين · طلبات الانضمام |
| التشغيل | طلبات الخدمة · الحملات · التعاونات · المحتوى والموافقات · العقود |
| المراجعات والامتثال | مراجعة العلامات · مراجعات العملاء |
| المالية | المستحقات |
| البيانات والتقارير | التقارير |
| الإدارة | الإعدادات (قريبًا) |

### العميل (client)
الرئيسية · العلاقات (ملف العميل · العلامات · العناوين · المستندات · الفريق) · التشغيل (طلبات الخدمة · الحملات · المحتوى والموافقات · العقود) · الحساب (الإشعارات · الملف المالي · الإعدادات)

### الشريك (partner)
الرئيسية · التشغيل (طلبات الخدمة) · الإدارة (البريفات · المحتوى · التقارير · الفريق · الإعدادات — قريبًا)

## 4. عدّادات الشارات (أرقام حقيقية، لا تجميلية)

| المفتاح | المصدر (PostgreSQL، مُنطّق بالمستأجر) |
|--------|--------------------------------------|
| `service_requests` | طلبات مفتوحة `whereIn(status, OPEN_STATUSES)` |
| `creator_applications` | طلبات انضمام `submitted/under_review` |
| `content` | محتوى بانتظار مراجعة الوكالة `status = agency_review` |
| `brand_reviews` | علامات `submitted/under_review` |
| `client_reviews` | تعديلات ملف معلّقة + مستندات `pending` |
| `client_notifications` | إشعارات العميل غير المقروءة (يُمرَّر عبر خاصية) |

القواعد:
- **تُحذف الأصفار** — الشارة تظهر فقط عند وجود عمل معلّق يحتاج إجراءً.
- كل عدّاد محميّ بـ `try/catch` → لا يُعطّل القائمة أبدًا.
- يُحسب مرّة واحدة لكل طلب (ذاكرة ثابتة داخل الطلب).

## 5. الجوال (Mobile)

- التنقّل السفلي الثابت يعرض حتى 5 وجهات (`mobile => true`).
- نقطة حمراء صغيرة تشير لوجود شارة (بدل الرقم لتوفير المساحة).
- أهداف لمس ≥ 56px ارتفاعًا، مع `env(safe-area-inset-bottom)`.
- المحتوى يُفسح له مكان أسفل الشريط عبر `.ih-shell.has-bottom-nav > main`.

## 6. الوصولية (a11y)

- العنصر النشِط يحمل `aria-current="page"`.
- الأيقونات `aria-hidden="true"` (زخرفية؛ النص هو المصدر).
- الشارة تحمل `aria-label` بعدد العناصر المعلّقة.
