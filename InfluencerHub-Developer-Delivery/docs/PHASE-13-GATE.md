# Phase 13 — محرّك الأتمتة/SLA (Automation / SLA Engine) — بوابة القبول

الحالة: **PHASE COMPLETE** (Feature Complete + مُتحقَّق بالمتصفّح + اختبارات خلفية). داخلي بالكامل (لا اعتمادات خارجية). التالي: Phase 14.

## ما أُنجز
- **البنية** `app/Domain/Automation`: `SlaEngineService` + `AutomationLog` + أعمدة تتبّع (`sla_reminded_at`/`sla_breached_at`) على `service_requests`.
- **قواعد مثبَّتة (dedup مرة واحدة لكل حدث)**:
  - **تذكير SLA**: طلب مفتوح يقترب من الاستحقاق (خلال نافذة 12 ساعة) ولم يُذكَّر → إشعار + سجلّ.
  - **رصد تجاوز SLA**: طلب مفتوح تجاوز الاستحقاق ولم يُرصَد → علَم `sla_breached_at` + إشعار + سجلّ + تدقيق.
- **الإشعار**: للمُسنَد إليه إن وُجد، وإلا لمديري الوكالة/العمليات في نفس المستأجر.
- **الجدولة**: أمر `sla:scan` مجدول **كل ساعة** (`withoutOverlapping`)، يعمل بتجاوز السياق ويجمّع لكل المستأجرين مع عزل تام.
- **السطح**: لافتة تحذير «تجاوز SLA (رُصد آليًا)» في تفصيل طلب الخدمة.

## الأمان (اختبارات + حيًّا)
- عزل مستأجر تام في المسح (إشعار المستأجر المعني فقط — مؤكَّد باختبار).
- Idempotent: لا رصد/تذكير مكرّر (dedup عبر أعمدة التتبّع).
- المغلق/المُلغى/غير المستحق لا يُرصد.

## الاختبارات
- **خلفي: 342 إجمالًا**. `SlaEngineTest(7)`: رصد+إشعار+سجلّ، Idempotency، تذكير واحد، لا إجراء للبعيد، تجاهل المغلق، إشعار المدراء عند عدم الإسناد، عزل مستأجر.
- **حيًّا**: ضبط SR-1-1 متأخّرًا → `php artisan sla:scan` (breaches=1) → لافتة «تجاوز SLA (رُصد آليًا)» ظهرت في التفصيل.
