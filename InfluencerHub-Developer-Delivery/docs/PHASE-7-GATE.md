# Phase 7 — منشئ الحملات (Campaign Builder) — بوابة القبول

الحالة: **PHASE COMPLETE** (Feature Complete + مُتحقَّق بالمتصفّح + اختبارات خلفية). التالي: Phase 8.

## ما أُنجز
- **البنية** `app/Domain/Campaigns`: `campaigns` + `campaign_deliverables` + `campaign_status_history`؛ `CampaignWorkflowService` (آلة حالة بالأحداث + مخرجات + تحويل من طلب).
- **آلة الحالة**: draft→planning/cancelled؛ planning→active/cancelled؛ active→paused/completed/cancelled؛ paused→active/cancelled. **حارس**: لا تُفعَّل حملة بلا مخرجات.
- **الميزانية بوحدات صغرى** (لا float) + عملة؛ حساب "الملتزَم" = Σ(أجر × كمية)؛ تنبيه تجاوز الميزانية.
- **تحويل من طلب**: طلب خدمة من نوع «حملة» → حملة (مسودة) مرتبطة بالمصدر (source_request_id)، idempotent، يرفض غير طلبات الحملات.
- **المخرجات**: نوع/منصّة/كمية/مبدع/أجر/تاريخ تسليم + حالة؛ إسناد تلقائي عند تحديد مبدع؛ تُدار فقط في draft/planning.
- **الوكالة** `/app/campaigns`: قائمة/إنشاء/تفصيل/مخرجات/آلة حالة + تحويل من طلب.
- **العميل** `/client/campaigns` (عرض فقط): يرى حملاته غير المسودة، مخرجاتها (نوع/منصّة/كمية/حالة) **دون الأجور/المبدعين** (تُدار من الوكالة).

## الأمان (اختبارات + حيًّا)
- عزل مستأجر + IDOR-safe؛ العميل لا يرى المسودات ولا الأجور.
- آلة حالة صارمة؛ العلامة يجب أن تخص العميل؛ التحويل idempotent.

## الاختبارات
- **خلفي: 289 إجمالًا**. `CampaignTest(12)`: رقم+مسودة، منع تفعيل بلا مخرجات، دورة كاملة، إسناد+ملتزَم، منع إضافة بعد التفعيل، منع انتقال غير صالح، تحويل من طلب (+idempotent)، رفض غير الحملة، إنشاء HTTP، عزل، رؤية العميل (بلا أجور)، منع رؤية المسودة.
- **حيًّا**: تحويل SR-1-1 → CM-1-1 (مرتبطة) → إضافة مخرج (ريل×3، مبدع، 4,500 → ملتزَم 13,500) → تخطيط → تفعيل (نشطة) + سجل حالة.
