# Product Execution Rules — InfluencerHub (دائمة، أعلى أولوية)

اقرأ هذا الملف مع [CONTINUATION-STATE.md](CONTINUATION-STATE.md) و[PRODUCT-COMPLETENESS-MATRIX.md](PRODUCT-COMPLETENESS-MATRIX.md) قبل كل مهمة. عند التعارض: **الجودة والاكتمال وعدم التراجع أعلى أولوية**.

## النمط
تصرّف كفريق منتج+تصميم+هندسة متكامل. لكل صفحة: افهم الهدف التجاري والمستخدمين والقرارات والأعمال؛ افحص Models/Relations/Routes/Controllers/Services/Policies/Workflows؛ اكتشف البيانات والوظائف غير المستغلة والنواقص؛ صمّم تجربة أصلية؛ نفّذ + اختبر + راجع بصريًا + حسّن عدّة جولات؛ ثم انتقل تلقائيًا. **لا تنتظر أفكارًا مني ولا أسأل عن الصفحة التالية.** اطلب تدخّلي فقط عند: قرار تجاري لا يُستنتج بأمان، بيانات اعتماد خارجية، تكلفة مالية، إجراء غير قابل للتراجع.

## الحزمة التقنية (ثابتة)
Laravel + PostgreSQL + Inertia.js + React + TypeScript + Vite. حافظ كاملًا على الـBackend/البيانات/المصادقة/الصلاحيات/العزل/Policies/Workflows/Queues/Notifications/Billing/Subscriptions/Entitlements/Integrations/الاختبارات. أبقِ Blade في `/app` بالتوازي حتى تجتاز نسخة React في `/beta` بوابة القبول، ثم انقل المسار وأزل Blade (لا نسختا منطق). ممنوع HTML ثابت أو SPA منفصل أو إعادة بناء من الصفر.

## كل صفحة تجيب: أين أنا؟ الحالة؟ أهم المعلومات؟ ما يحتاج انتباهي؟ المخاطر/النواقص؟ الإجراء الآن؟ الخطوة التالية؟ المسؤول؟ الموعد؟ ما تغيّر؟ الكيانات المرتبطة؟ كيف أفتح/أنفّذ؟

## الطبقات الثلاث لكل صفحة رئيسية
1. **الفهم:** هوية الوحدة + ملخص + حالة + مؤشرات + تقدّم + جاهزية + مخاطر + نواقص + سياق.
2. **التنفيذ:** إجراءات رئيسية/سياقية + بحث + فلاتر + Saved Views + تحديد متعدد + Bulk + Quick Preview + Inline + مهام/موافقات/ملفات/تعليقات + تنقّل للكيانات.
3. **الذكاء التشغيلي:** Next Best Action + Readiness/Health + Risk + SLA + تأخّر + جودة بيانات/اكتمال + تعارضات + فرص + تنبيهات قابلة للتنفيذ + ملخّص تغييرات + توصيات Rule-Based. **لا مؤشرات شكلية بلا إجراء.**

## الاكتمال الوظيفي (افحص الحاجة، نفّذ ما يخدم الرحلة فقط)
إنشاء/عرض/تعديل/أرشفة/استعادة/بحث/تصفية/ترتيب/Pagination/استيراد/تصدير/تحديد متعدد/إجراءات جماعية/ملاحظات/مهام/ملفات/نشاط/Timeline/موافقات/إشعارات/صلاحيات/سجل رقابي/علاقات/حالات خطأ+فارغة+تحميل/جوال.

## الصدق (لا قدرات وهمية)
لا تدّعِ API/دفع/تحليلات/بيانات منصّة/تحقّق خارجي غير متصل. استخدم حالات صريحة: Manual · Sandbox · Waiting for Credentials · Waiting for Approval · Connected · Degraded · Unavailable.

## الإبداع = تنظيم أذكى للمعلومات، خطوات أقل، إبراز ما يحتاج تدخّلًا، ربط البيانات، وضوح الإجراءات، Workspaces متخصّصة، إظهار المخاطر مبكرًا، رؤية شاملة بلا ازدحام، جوال حقيقي. ليس Gradient/ظلال/حركات بلا هدف. Smart Code حدّ أدنى للنضج — لا نسخ هوية/كود/لون/ترتيب.
