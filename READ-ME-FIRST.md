# InfluencerHub — اقرأني أوّلًا

منصّة إدارة التسويق مع صنّاع المحتوى: من طلب الحملة إلى تسوية المستحقّات.
عربية أوّلًا · RTL · متعدّدة المستأجرين.

> **هذه حزمة مصدر.** لا تحوي `vendor` ولا `node_modules` ولا `.env` ولا قاعدة
> بيانات. ملفّات القفل مضمَّنة، فتُثبَّت الإصدارات نفسها بالضبط.

---

## 1. البنية التقنية

| العنصر | الإصدار |
|---|---|
| PHP | **8.4** (الحدّ الأدنى في `composer.json`: 8.2) |
| Laravel | **12** |
| قاعدة البيانات | **PostgreSQL 16** — إلزامية |
| Node | **24** (يعمل على 20+) |
| الواجهة | Inertia 3 + React 19 + TypeScript 7 + Vite 7 |
| الاختبارات | PHPUnit — **1043 اختبارًا** |

### لماذا PostgreSQL إلزامية

ليست تفضيلًا: الشيفرة تستعمل **فهارس جزئية** (`CREATE UNIQUE INDEX … WHERE`)
لمنع تكرار طلبات المطالبة، وتستعمل `ilike` في البحث. لا تعمل على MySQL ولا
SQLite. وقاعدة الاختبارات منفصلة.

### Redis

مُهيَّأ في `config/` وغير مستعمَل: كل السائقين (`cache` · `session` · `queue`)
على `database`. **لا تُبدّله قبل قياس الحاجة** — التبديل بلا قياس يضيف عطلًا
تشغيليًّا بلا مكسب مُثبَت.

## 2. التشغيل من الصفر

```bash
# 1) الاعتماديات — بملفّات القفل، فتُثبَّت الإصدارات نفسها
composer install
npm ci

# 2) البيئة
cp .env.example .env
php artisan key:generate

# 3) قاعدة البيانات
createdb influencerhub
createdb influencerhub_testing          # لازمة للاختبارات
#    ثم عدّل DB_USERNAME/DB_PASSWORD في .env

php artisan migrate

# 4) الواجهة
npm run build                            # أو: npm run dev للتطوير الحيّ

# 5) التشغيل
php artisan serve --host=127.0.0.1 --port=8010
```

افتح <http://127.0.0.1:8010>.

### بذور آمنة (محلّي فقط)

```bash
PREVIEW_PASSWORD=<كلمة-مرور-محلّية> php artisan preview:seed   # حسابات معاينة
E2E_PASSWORD=<كلمة-مرور-محلّية>     php artisan e2e:seed       # مستأجران للاختبار
php artisan creators:seed-categories                            # كتالوج فئات
```

كلاهما **يرفض العمل في بيئة الإنتاج**. لا تُشغّلهما على قاعدة حقيقية.

### العامل والمجدوِل

```bash
php artisan queue:work --queue=default --tries=3
php artisan schedule:work        # محلّيًّا
# على الخادم: * * * * * cd /path && php artisan schedule:run >> /dev/null 2>&1
```

المجدوِل يشغّل `sla:scan` (رصد تجاوز المواعيد) — انظر `routes/console.php`.

### الاختبارات

```bash
php artisan test                 # 1043 اختبارًا
npx tsc --noEmit                 # يجب أن يكون نظيفًا
./scripts/check-tenant-context-safety.sh   # حارس عزل المستأجرين
```

> ⚠️ **اسم قاعدة الاختبار مثبَّت.** `ConcurrentClientCreationTest` يُشغّل
> عمليّتين فرعيّتين ويمرّر لهما `DB_DATABASE=influencerhub_testing` **نصًّا
> صريحًا**. فإن سمّيت قاعدة اختبارك باسم آخر، فشل هذا الاختبار وحده بينما
> ينجح 1042 غيره — وهو فشلٌ مُربك سببه الاسم لا الشيفرة. استعمل الاسم نفسه،
> أو عدّل السطر 45 من الاختبار.

> ⚠️ **لا تُشغّل مجموعتَي اختبار متوازيتين.** كلتاهما تهاجر
> `influencerhub_testing` فيقع تعارض في PostgreSQL وتظهر إخفاقات وهمية متغيّرة.

## 5. الأرقام — لاتينية دائمًا

**قاعدة معتمَدة:** كل رقم يظهر للمستخدم أو يُكتب في وثيقة يكون لاتينيًّا
(`0123456789`) لا هنديًّا (`٠١٢٣٤٥٦٧٨٩`).

والموضع الذي يخالفها **بصمت** هو التنسيق وقت التشغيل:

```js
new Intl.NumberFormat('ar-SA', …)   // ❌ يُخرج ٣٢٬٤٠٠
new Intl.NumberFormat('en-US', …)   // ✅ يُخرج 32,400
```

لا يظهر هذا في مراجعة الشيفرة — `'ar-SA'` تبدو صوابًا في تطبيق عربي — وإنّما
يظهر في الصفحة وحدها. استعمل `'en-US'` في كل تنسيق رقمي، وهو ما تفعله بقيّة
الملفّات (55 ملفًّا).

للفحص:

```bash
git grep -lE '٠|١|٢|٣|٤|٥|٦|٧|٨|٩'   # يجب أن يعود فارغًا
git grep -n "NumberFormat('ar"        # يجب أن يعود فارغًا
```

## 6. العزل بين المستأجرين — اقرأ هذا قبل أيّ استعلام

`TenantContext` + `TenantScope` بنمط **fail-closed**: بلا سياق يعود الاستعلام
**فارغًا لا خطأً**. والصمت هو العطل — نتيجةٌ فارغة تعني «لا سياق» بقدر ما تعني
«لا بيانات».

- استعمل `TenantContext::withTenant()` و`withBypass()` — تستعيدان ما كان حتّى
  عند الاستثناء.
- **`set()`/`reset()`/`bypass()` اليدوية ممنوعة** في `app/`، ويفشل البناء
  عليها. الاستثناء ستّة ملفّات بالاسم والسبب في السكربت.
- التفصيل الكامل وتسعة أعطال وقعت فعلًا: [`docs/TENANT-CONTEXT-SAFETY.md`](docs/TENANT-CONTEXT-SAFETY.md).

```bash
./scripts/check-tenant-context-safety.sh    # يجب أن يبقى أخضر
```

## 7. ما هو مكتمل

- **التعدّدية والهوية**: مستأجرون · مؤسّسات · عضويات · أدوار · عزل مُختبَر.
- **الاشتراكات والحقوق**: خطط بإصدارات · استحقاقات · قياس استهلاك وحدوده.
- **CRM**: عملاء · علامات · جهات اتصال · مستندات · عناوين.
- **المبدعون**: ملفّات · قدرات مزدوجة · طلبات انضمام · بوّابة · دعوات وربط حساب.
- **الرحلة التشغيلية كاملة**: طلب ← حملة ← ترشيح ← تعاون ← عقد ← محتوى
  (دورتا مراجعة) ← إثبات نشر ← فاتورة ← مستحقّ ← إغلاق.
- **المالية**: فواتير · مستحقّات · معادلة ربح واحدة (`FinancialMetrics`).
- **الشركاء**: وكالات خارجية · أعضاء · ربط مُنطّق بالعميل.
- **العلامة ككيان مستقلّ**: تسجيل ذاتي · مطابقة موزونة · مطالبة بعلامة قائمة ·
  مساحة عمل · تفويض وكالة بنطاق مع إلغاء بلا فقد بيانات.

## 8. ما هو غير مكتمل

| البند | الحال |
|---|---|
| **لوحة مراجعة المطالبات** | الخدمة كاملة ومُختبَرة (`BrandClaimService`) — **بلا واجهة**. المراجعة برمجيًّا فقط. **أهمّ نقص.** |
| توحيد التحقّق بالجوال | تدفّق العلامة يتحقّق من البريد والجوال؛ الوكالة والمبدع من البريد وحده |
| الإنشاء من داخل مساحة العلامة | الأقسام تعرض وتقرأ؛ الإنشاء عبر مسار الوكالة |
| قبول التفويض من واجهة الوكالة | الخدمة تدعمه، الزرّ غير مبنيّ |
| جدولة انتهاء المطالبات | `expireDue()` موجودة ومُختبَرة وغير مجدوَلة |
| `/admin` | غير مبنيّ — الموجود `/beta/admin` |
| قدرات دور العلامة | انظر §4 |
| **CI** | **لا يوجد.** الحارس والاختبارات لا تعمل تلقائيًّا عند الدفع |
| التكاملات | لا مزوّد مربوط — [`docs/API-INTEGRATIONS.md`](docs/API-INTEGRATIONS.md) |

**نقطة الاستئناف الدقيقة**: [`docs/CONTINUATION-STATE.md`](docs/CONTINUATION-STATE.md).

## 9. التكاملات المنتظِرة اعتمادًا

**لا تكامل خارجيّ واحد مربوط.** البريد `log` محلّيًّا، والرسائل النصّية بحالة
`WAITING_FOR_CREDENTIALS`، و**النظام لا ينفّذ أيّ تحويل مالي** — «مدفوع»
تُسجَّل يدويًّا بمرجع بعد تسوية حقيقية.

القاعدة: ممنوع ادّعاء مزامنة أو دفع حقيقي بلا اعتماد رسمي.

التفصيل: [`docs/EXTERNAL-BLOCKERS.md`](docs/EXTERNAL-BLOCKERS.md) ·
[`docs/API-INTEGRATIONS.md`](docs/API-INTEGRATIONS.md).
