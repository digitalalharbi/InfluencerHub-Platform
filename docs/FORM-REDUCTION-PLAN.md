# Form Reduction Plan — InfluencerHub

خطة تقليل الاحتكاك في النماذج. الهدف: أقل عدد حقول في الخطوة الأولى، والباقي مُشتق/مؤجّل/مستورد.

## 1. تصنيف كل حقل

| الفئة | التعريف | المعالجة |
|------|--------|----------|
| **ضروري الآن** | لا يمكن إنشاء الكيان بدونه | يظهر في الخطوة الأولى |
| **قابل للاشتقاق** | يُحسب من بيانات أخرى | لا يُطلب — يُحسب تلقائيًا |
| **قابل للتأجيل** | مهم لكن ليس للإنشاء | Progressive Disclosure / لاحقًا في المساحة |
| **قابل للاستيراد** | متوفّر في مصدر آخر | استيراد بدل إدخال يدوي |
| **افتراضي ذكي** | له قيمة شائعة | يُملأ مسبقًا وقابل للتعديل |

## 2. أمثلة تطبيقية

| الكيان | ضروري الآن | مؤجّل/مشتق |
|-------|-----------|-----------|
| عميل | الاسم + النوع | الرقم (يُشتق تلقائيًا CL-x)، العناوين/المستندات/الفريق (لاحقًا في المساحة) |
| حملة | العنوان + العميل | الميزانية/المخرجات (تُضاف تدريجيًا)، العملة (افتراضي SAR) |
| طلب خدمة | النوع + العنوان | الأولوية (افتراضي)، الإسناد/المهلة (في الفرز) |
| مبدع | الاسم + المنصّة | الفئات/المحفظة (لاحقًا) |

## 3. أنماط

- **Quick Create:** زر إنشاء سريع يفتح نموذجًا مصغّرًا (ضروري الآن فقط) ثم يفتح المساحة لإكمال الباقي.
- **الافتراضات الذكية:** العملة SAR، الأولوية «عادية»، الحالة الأولية «مسودة» — كلها قابلة للتعديل.
- **الاشتقاق التلقائي:** الأرقام المرجعية (CL-/CMP-/SR-) تُولّد خادميًا، لا تُطلب.
- **الكشف التدريجي:** الحقول المتقدّمة خلف «خيارات إضافية».

## 4. قاعدة صارمة

- لا تُطلب بيانات يمكن اشتقاقها أو استيرادها.
- كل حقل مطلوب يجب أن يجيب: «هل يمنع غيابه إنشاء الكيان الآن؟» إن لا → أجّله.

## 5. الحالة

- البنية (Quick Create، الافتراضات، الاشتقاق الخادمي للأرقام) قائمة في الدومين.
- التطبيق التدريجي على النماذج الأطول: راجع [EXPERIENCE-AUDIT](EXPERIENCE-AUDIT.md) §2.
