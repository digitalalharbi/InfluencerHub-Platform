# Phase 10 — العقود (Contracts) — بوابة القبول

الحالة: **PHASE COMPLETE** (Feature Complete + مُتحقَّق بالمتصفّح + اختبارات خلفية). التالي: Phase 11.

## ما أُنجز
- **البنية** `app/Domain/Contracts`: `contracts` + `contract_status_history`؛ `ContractWorkflowService` (دورة حياة بالأحداث).
- **دورة الحياة**: draft→sent→signed→active→completed؛ +terminate (من active) / cancel. سجل append-only يميّز الفاعل.
- **الطرف مبدع أو عميل**: العقد يستهدف مبدعًا أو عميلًا (اختياريًا مرتبط بتعاون/حملة)، قيمة بوحدات صغرى + عملة.
- **القبول داخل البوابة (صادق)**: الطرف المقابل يقبل داخل بوابته — يُسجَّل **اسم + وقت + مستخدم**، مع تنويه صريح أنّه **قبول داخل المنصّة وليس توقيعًا قانونيًا خارجيًا**.
- **الوكالة** `/app/contracts`: إنشاء/تحرير البنود (مسودة فقط)/إرسال/تفعيل/إكمال/إنهاء.
- **المبدع** `/creator/contracts` + **العميل** `/client/contracts`: عرض العقود المُرسَلة+ وقبولها (لا المسودات). قبول العميل لمدير العميل فقط.
- **إشعارات**: إرسال العقد → الطرف؛ قبول الطرف → الوكالة.

## الأمان (اختبارات + حيًّا)
- عزل مستأجر + IDOR-safe (الطرف يرى/يقبل عقوده فقط، لا المسودات).
- لا تحرير بعد الإرسال؛ آلة حالة صارمة.
- قبول العميل مقيّد بـclient_admin.

## الاختبارات
- **خلفي: 321 إجمالًا**. `ContractTest(10)`: إنشاء+رقم، اشتراط الطرف، دورة كاملة (إرسال/قبول/تفعيل/إكمال)، منع التحرير بعد الإرسال، منع انتقال غير صالح، قبول المبدع HTTP، منع رؤية المسودة، قبول العميل HTTP، منع IDOR توقيع، عزل مستأجر.
- **حيًّا**: الوكالة تنشئ CT-1-1 (Renad، 13,500) → إرسال → المبدع يراه (مع تنويه القبول داخل المنصّة) → يقبل (مقبول، سُجّل «رينـاد الزهراني» + وقت).

## التالي
Phase 11 (الشحن والهدايا / المالية) وما بعدها.
