# Phase 6 — طلبات الخدمة الخارجية (External Service Requests) — بوابة القبول

الحالة: **PHASE COMPLETE** (Feature Complete + مُتحقَّق بالمتصفّح + اختبارات خلفية). التالي: Phase 7 (منشئ الحملات).

> تسمية صادقة: مكتملة الميزات ومُختبَرة ومُعايَنة حيًّا عبر البوابات الثلاث. ليست "PRODUCTION READY".

## ما أُنجز
- **البنية** `app/Domain/Requests`: `service_requests` + `service_request_comments` (داخلي/خارجي) + `service_request_status_history` (append-only)، تعدادات النوع/الأولوية، `ServiceRequestWorkflowService` (آلة حالة بالأحداث + SLA).
- **آلة الحالة** (لا Dropdown يدوي): submitted→triage/cancelled؛ triage→in_progress/needs_info/cancelled؛ in_progress→needs_info/resolved/cancelled؛ needs_info→in_progress/cancelled؛ resolved→closed/in_progress(إعادة فتح).
- **SLA**: `due_at` يُحسب من الأولوية (urgent 4h / high 24h / normal 72h / low 168h)؛ مؤشّر تأخّر (overdue) في القائمة.
- **العميل** `/client/requests`: إنشاء (لعميله)، قائمة، تفصيل، محادثة (لا يرى الملاحظات الداخلية).
- **الشريك** `/partner/requests`: إنشاء **مقيّد بالعملاء المرتبطين فقط** (fail-closed 422 لغير المرتبط)، قائمة/تفصيل/محادثة.
- **الوكالة** `/app/service-requests`: قائمة بفلاتر (حالة/أولوية) + SLA، تفصيل، إسناد لعضو (من نفس المستأجر)، إجراءات الحالة، تعليقات خارجية/داخلية، سجل حالة.
- **إشعارات**: تغيّر الحالة → إشعار للمُرسِل (عبر NotificationService)؛ رد جديد → إشعار للمُسنَد إليه.

## الأمان (مُتحقَّق باختبارات + حيًّا)
- عزل مستأجر fail-closed + IDOR-safe (العميل يرى طلبات عميله فقط؛ الشريك طلبات وكالته فقط).
- **رؤية التعليقات**: الملاحظات الداخلية (is_internal) لا تظهر للعميل/الشريك — مؤكَّد حيًّا وباختبار.
- الشريك لا ينشئ طلبًا إلا لعميل ضمن روابطه النشطة.
- الإسناد مقيّد بأعضاء نفس المستأجر.
- انتقالات الحالة عبر آلة حالة صارمة (منع الانتقالات غير الصالحة).

## الاختبارات
- **خلفي: 277 اختبارًا** ناجحة إجمالًا. `ServiceRequestTest(10)`: SLA+رقم، مسار كامل، منع انتقال غير صالح، إعادة فتح، إنشاء العميل عبر HTTP، إخفاء الداخلية، عزل IDOR للعميل، تقييد الشريك بالمرتبط، إسناد+انتقال الوكالة، عزل مستأجر.
- **حيًّا (متصفّح Claude)**: العميل ينشئ SR-1-1 (حملة/عالية) → الوكالة تراه بSLA → ملاحظة داخلية (مخفية) + فرز + رد خارجي → العميل يرى الحالة الجديدة والرد الخارجي فقط + شارة إشعار.

## التالي
Phase 7 (منشئ الحملات) — تحويل طلبات الحملات إلى حملات كاملة، ثم بقية المراحل.
