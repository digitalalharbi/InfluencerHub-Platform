# Phase 9 — المحتوى والموافقات (Content & Approvals) — بوابة القبول

الحالة: **PHASE COMPLETE** (Feature Complete + مُتحقَّق بالمتصفّح + اختبارات خلفية). التالي: Phase 10.

## ما أُنجز
- **البنية** `app/Domain/Content`: `content_items` + `content_approvals` (قرار لكل مرحلة) + `content_status_history`؛ `ContentWorkflowService` (سير موافقات متعدّد المراحل بالأحداث).
- **خط الموافقات**: draft→submitted→agency_review→(الوكالة) client_review/changes_requested/rejected→(العميل) approved/changes_requested/rejected→scheduled/published. إعادة التقديم بعد طلب التعديل **تزيد الإصدار**. سجل append-only يميّز الفاعل (مبدع/وكالة/عميل)، وقرارات مسجّلة لكل مرحلة.
- **المبدع** `/creator/content`: إنشاء (اختياريًا مرتبط بتعاون قيد التنفيذ)، تعديل (draft/changes_requested فقط)، تقديم، رؤية سبب التعديل.
- **الوكالة** `/app/content`: مراجعة، إرسال لموافقة العميل، طلب تعديل/رفض، جدولة/نشر.
- **العميل** `/client/content`: يرى المحتوى في مرحلة مراجعته وما بعدها فقط (لا المسودات/مرحلة الوكالة)؛ اعتماد/طلب تعديل (client_admin/content_reviewer).
- **إشعارات**: طلب تعديل/اعتماد → المبدع.

## الأمان (اختبارات + حيًّا)
- عزل مستأجر + IDOR-safe (المبدع محتواه فقط؛ العميل محتوى عميله وبعد client_review فقط).
- آلة حالة صارمة؛ المبدع لا يعدّل بعد التقديم.
- صلاحيات اعتماد العميل مقيّدة بالأدوار.

## الاختبارات
- **خلفي: 311 إجمالًا**. `ContentTest(11)`: إنشاء+رقم، خط كامل حتى النشر (قراران)، طلب تعديل الوكالة→إعادة تقديم=إصدار جديد، طلب تعديل العميل، منع انتقال غير صالح، منع تعديل بعد التقديم، تقديم المبدع HTTP، اعتماد العميل HTTP، منع رؤية العميل قبل client_review، IDOR مبدع، عزل مستأجر.
- **حيًّا**: Renad ينشئ CN-1-1 → يقدّمه (مُقدَّم) → الوكالة تبدأ المراجعة → إرسال للعميل (مراجعة العميل + قرار «الوكالة موافقة» مسجّل).
