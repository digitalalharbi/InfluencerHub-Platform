## 3) الأمان

- ✅ لا أسرار في Git (`git ls-files` نظيف؛ `.env` وملفات الاعتماد وقياسات الأداء مُتجاهَلة).
- ✅ الملفات الخاصة على قرص `local` — لا رابط عام ثابت؛ التنزيل يمرّ بالخادم ويُسجَّل.
- ✅ الآيبان مُشفَّر (`FinancialCrypto`) ولا يُعاد إلا بآخر أربعة أرقام.
- ✅ رموز الدعوة تُعرض مرّة واحدة ولا تُخزَّن خامًا.
- ✅ تغيير كلمة المرور يُبطل الجلسات الأخرى.
- ✅ الصلاحيات مغلقة افتراضيًا (deny-by-default) في `CrmAbilities`/`ClientPortalAbilities`.
- ✅ منع IDOR مُختبَر على: مستندات، عناوين، منصّات/خدمات المبدع، مخرجات الحملات، الإشعارات.
- ⛔ **التحقّق بخطوتين (2FA)**: الحقل موجود (`two_factor_confirmed_at`) والحالة تُعرض، لكن تدفّق التفعيل غير مبنيّ. قرار منتَج.

## 4) الفوترة والمدفوعات — ⛔ عائق خارجي

**الحالة الصادقة: لا مدفوعات حقيقية في النظام.**

| البند | الحالة |
|---|---|
| الخطط/الإصدارات/الحقوق/الاستهلاك | ✅ مبني ومختبَر |
| الاشتراكات + أحداثها | ✅ مبني ومختبَر |
| حدود الخطة (Entitlements) | ✅ مُطبَّقة فعليًا (تمنع التجاوز) |
| مزوّد الدفع | ⛔ `FakeBillingProvider` فقط — الوحيد المُنفِّذ لعقد `BillingProvider` |
| الفواتير (Invoice) | ⛔ غير موجودة |
| المدفوعات (Payment) | ⛔ غير موجودة |
| دفتر الأستاذ (Ledger) | ⛔ غير موجود |
| Webhooks الفوترة | ⛔ غير موجودة |

**لا يجوز إطلاق فوترة تجارية بهذه الحالة.** المطلوب قرار تجاري + اعتمادات:
اختيار المزوّد (ميسر/Stripe)، حساب تاجر، مفاتيح، وسياسة ضريبة القيمة المضافة والفوترة الإلكترونية (فاتورة/ZATCA).
المستحقات للمبدعين تُسجَّل يدويًا بمرجع بعد تسوية حقيقية — النظام لا ينفّذ تحويلًا.

## 5) التكاملات — حالات صادقة

| المنصّة | الحالة الفعلية |
|---|---|
| Snapchat · TikTok · X · LinkedIn · YouTube · Instagram | `available_manual` — إدخال/استيراد يدوي |
| Threads | `draft` — مخفيّة تمامًا |
| الاكتشاف الحيّ عبر APIs | ⛔ `WAITING_FOR_CREDENTIALS` |
| Google Drive | ⛔ **غير مبنيّ إطلاقًا** — لا كود ولا إعداد. التخزين محلي خاص |
| الإعلانات المدفوعة | ⛔ `WAITING_FOR_CREDENTIALS` |
| البريد/SMS | ✅ بريد فعلي · ⛔ SMS `NullSmsSender` بحالة انتظار |

**لا موصّل يدّعي `connected` بلا اعتماد فعلي.** الحالة تُرفَّع تلقائيًا عند توفّر الاعتماد.
