# Responsive Design Standard — InfluencerHub

المعيار التجاوبي الموحّد. مبني على **Mobile-First** مع قاعدة صارمة واحدة فوق كل شيء.

## 0. القاعدة الصارمة

> **لا تمرير أفقي للصفحة إطلاقًا.** على أي مقاس: `document.documentElement.scrollWidth ≤ clientWidth`.

المحتوى العريض (جداول، أكواد، مخططات) يُمرَّر **داخل حاويته** (`overflow-x:auto`)، لا عبر الصفحة.
مفروضة عالميًا في `app.css`: `html, body { overflow-x: hidden; max-width: 100% }` + `img,svg,video,table { max-width: 100% }`.

## 1. نقاط الكسر (Breakpoints)

| الاسم | العرض | الاستخدام |
|------|------|-----------|
| sm | 375px | جوال |
| md | 768px | لوحي / حد تبديل الدرج |
| lg | 1024px | لوحي عريض |
| xl | 1280px | سطح مكتب |
| 2xl | 1440px | شاشة واسعة |

معرّفة كـ tokens في `tokens.css` (`--ih-bp-*`).

## 2. هيكل التطبيق (App Shell)

- **سطح المكتب (≥768px):** شريط جانبي ثابت + رأس علوي.
- **الجوال (<768px):** الشريط الجانبي يصبح **درجًا** (`position:fixed` خارج الشاشة، يفتح بزر القائمة)، مع طبقة تعتيم (scrim) وشريط علوي للجوال. الرأس المكتبي يُخفى.
- **التنقّل السفلي** الثابت (4–5 وجهات) يظهر على الجوال فقط.

## 3. المساحات الآمنة (Safe Areas)

- `<meta name="viewport" content="... viewport-fit=cover">` في كل التخطيطات.
- ارتفاعات ديناميكية: `100dvh`/`100svh` بدل `100vh`.
- حشو يحترم `env(safe-area-inset-*)` عبر tokens `--ih-safe-*` (أعلى الشريط، أسفل التنقّل السفلي، جوانب auth).

## 4. راحة اللمس (Touch)

- أهداف اللمس ≥ 44px (أزرار)، عناصر القائمة ≥ 44px، التنقّل السفلي ≥ 56px.
- خط الإدخال ≥ **16px** على الجوال لمنع **تكبير iOS التلقائي**، ويتقلّص إلى 0.9rem من 768px.

## 5. الجداول التجاوبية

- ≤640px: البطاقة `overflow-x:auto`، والجدول `min-width:520px` يُمرَّر داخلها فقط.
- الرؤوس والخلايا بحشو أصغر وخط 0.82rem.

## 6. الحركة

- `@media (prefers-reduced-motion: reduce)` يعطّل الانتقالات فعليًا.

## 7. الإنفاذ الآلي

`tests/e2e/16-responsive.spec.js` (Playwright):
- يفحص **كل مسار تمثيلي** لكل بوابة عبر 4 مقاسات (375/768/1280/1440) → لا تمرير أفقي.
- محاكاة جوال (390×844): التنقّل السفلي مرئي (3–5 روابط)، الدرج يفتح، خط الإدخال ≥ 16px.
