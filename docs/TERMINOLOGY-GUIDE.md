# Terminology Guide — InfluencerHub

دليل المصطلحات الموحّد. **مصدر واحد للحقيقة**: كل نص ظاهر للمستخدم يأتي من ملفات `lang/`، لا يُكتب في الواجهات مباشرة.

## 1. الملفات المركزية

| الملف | المحتوى |
|------|---------|
| `lang/{ar,en}/navigation.php` | عناوين المجموعات + تسميات عناصر القوائم + أوصافها |
| `lang/{ar,en}/entities.php` | أسماء الكيانات (مفرد/جمع) |
| `lang/{ar,en}/statuses.php` | قاموس الحالات + خريطة الحالة→اللون |
| `lang/{ar,en}/actions.php` | أفعال الإجراءات (موجّهة للنتيجة) |

الاستخدام في Blade: `{{ __('navigation.items.clients') }}` · `{{ __('statuses.under_review') }}` · `{{ __('actions.submit_for_review') }}`.

## 2. قواعد التسمية

1. **سمِّ الكيان، لا تصف الفعل.** ✅ «العملاء» ❌ «إدارة العملاء / إدارة البيانات».
2. **لا تسميات مبهمة.** ممنوع: «الإدارة»، «البيانات» كعنصر قائمة مفرد بلا سياق.
3. **مصطلح واحد لكل مفهوم عبر كل البوابات.** «طلبات الخدمة» تعني الشيء نفسه للوكالة والعميل والشريك.
4. **الإجراء يعبّر عن النيّة لا آلية القاعدة.** ✅ «إرسال للمراجعة» ❌ «تغيير الحالة إلى submitted».

## 3. تمييز الكيانات المتقاربة

| المصطلح | التعريف | ليس |
|---------|---------|-----|
| **المؤثر** (influencer) | صاحب جمهور على منصّات التواصل | ليس بالضرورة صانع UGC |
| **صانع المحتوى** (UGC creator) | ينتج محتوى للعلامة دون اشتراط جمهور كبير | ليس مؤثرًا بالضرورة |
| **الناشر** (publisher) | جهة نشر/وسيط توزيع | ليس فردًا |
| **العميل** (client) | الحساب المتعاقد مع الوكالة | ليس العلامة التجارية |
| **العلامة التجارية** (brand) | علامة مملوكة لعميل | قد يملك العميل عدة علامات |
| **الوكالة الشريكة** (partner) | وكالة خارجية متعاونة | ليست العميل |

## 4. قاموس الحالات (مقتطف)

| المفتاح | العربية | اللون (ih-status-\*) |
|--------|---------|---------------------|
| `draft` | مسودة | رمادي |
| `submitted` | مُرسل | أزرق (info) |
| `under_review` | قيد المراجعة | كهرماني (warning) |
| `changes_requested` | تعديلات مطلوبة | برتقالي |
| `approved` | معتمد | أخضر |
| `active` | نشِط | بنفسجي |
| `rejected` | مرفوض | أحمر (danger) |
| `completed` | مكتمل | أخضر |
| `archived` | مؤرشف | رمادي |

الخريطة الكاملة + ربط الحالة باللون في `lang/ar/statuses.php` تحت المفتاح `tone`.

## 5. الفحوصات الآلية

`tests/Feature/NavigationSystemTest.php` يضمن:
- كل عنصر قائمة له تسمية في **العربية والإنجليزية** (لا مفتاح خام ظاهر).
- قاموس الحالات **متماثل** بين اللغتين (نفس المفاتيح).
- كل مفتاح شارة معروف ومربوط بمصدر عدّ حقيقي.

## 6. الإضافة أو التعديل

1. أضِف المفتاح في `lang/ar/*.php` **و** `lang/en/*.php` معًا.
2. إن كان عنصر قائمة: أضِفه في `config/navigation.php` بالبوابة والمجموعة المناسبة.
3. استخدم `__('...')` في الواجهة — لا تكتب النص مباشرة.
4. شغّل `php artisan test --filter=NavigationSystemTest`.
