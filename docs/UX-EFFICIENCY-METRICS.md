# UX Efficiency Metrics — InfluencerHub

مؤشرات كفاءة التجربة: كيف نقيس أن المنتج «منخفض الاحتكاك».

## 1. المؤشرات

| المؤشر | التعريف | الهدف |
|-------|--------|------|
| **نقرات حتى الوجهة** | عدد النقرات للوصول لأي شاشة | ≤ 1 عبر لوحة الأوامر ⌘K |
| **حقول الخطوة الأولى** | عدد الحقول المطلوبة للإنشاء | الحد الأدنى المطلق (ضروري الآن فقط) |
| **زمن الإنشاء الأول** | من فتح النموذج إلى حفظ الكيان | مُقلَّل عبر Quick Create + الافتراضات |
| **إجراءات موجّهة مقابل قوائم status** | نسبة الأزرار الموجّهة للنتيجة | 100% (لا قوائم status يدوية) |
| **عمق التنقّل** | مستويات القائمة للوصول لوظيفة | ≤ 2 (مجموعة → عنصر) |
| **تغطية الشارات الحيّة** | وحدات ذات عدّاد عمل معلّق حقيقي | طلبات/انضمام/محتوى/مراجعات |

## 2. القياس الحالي (قابل للتحقّق)

| المؤشر | القيمة |
|-------|-------|
| نقرات حتى أي وجهة عبر ⌘K | 1 (فتح + كتابة + Enter) — مُختبر E2E |
| عمق التنقّل | 2 (مجموعة → عنصر) عبر كل البوابات |
| عدّادات حيّة من PostgreSQL | 6 مفاتيح (طلبات/انضمام/محتوى/مراجعة علامات/مراجعات عملاء/إشعارات) |
| إجراءات موجّهة | وحدات المراجعة/الطلبات/المحتوى تستخدم انتقالات بالأحداث لا قوائم status |
| لا تمرير أفقي | مُنفَذ آليًا عبر 3 بوابات × 4 مقاسات |

## 3. أدوات القياس

- **آليًا:** مجموعات Playwright (`16-responsive`, `17-command-palette`) تقيس التنقّل والتجاوب.
- **يدويًا:** مراجعة كل صفحة تفاصيل مقابل قائمة [EXPERIENCE-AUDIT](EXPERIENCE-AUDIT.md).

## 4. مبدأ

الكفاءة ليست «شاشات أقل» بل **احتكاكًا أقل لكل مهمة**: أقل حقول، أقصر مسار، إجراء أوضح، وسياق أغنى في مكان واحد.
